# Story Analyzer — 物語構造分析・開発ツール

4つの物語構造フレームワークを統合し、脚本・プロットの**診断**と**新規ストーリー開発**を支援するClaude Codeスキルです。

## 統合フレームワーク

| フレームワーク | 概要 |
|---|---|
| **Save the Cat** | Blake Snyderの15ビート。ハリウッド標準の構造テンプレート |
| **6つの感情アーク** | Reagan et al. (2016) の研究。1,327作品の分析から導出された6つの基本パターン |
| **三幕構成** | Setup / Confrontation / Resolution。西洋ドラマの基盤 |
| **起承転結** | 東洋の四部構成。マンガとの親和性が高い |

## Claude Codeスキルとして使う

### アーキテクチャ

| レイヤー | 役割 | ファイル |
|---|---|---|
| **AIスキル層** | 脚本の読解・フレームワーク分析・対話・ビートシート生成 | `SKILL.md` |
| **参照データ層** | 4フレームワークの定義・スコアリング基準 | `references/*.md` |
| **可視化層** | 感情アークグラフ+スコアカードのPNG出力 | `scripts/visualize_story.py` |

### セットアップ

```bash
# 1. リポジトリをクローン
git clone https://github.com/yama-kiyo/story-analyzer.git ~/.claude/skills/story-analyzer

# 2. 依存ライブラリをインストール
pip install matplotlib numpy

# 3. IPA Gothicフォントをインストール（未導入の場合）
brew install font-ipafont
```

### 2つのモード

#### 分析モード — 既存の脚本を診断する

```
ユーザー: この脚本を分析して [ファイルを渡す]

AI: フォーマットは映画ですね。分析します...

    検出された感情アーク: Cinderella型（感動）、一致度 87%

    【スコアカード】
    構造:     A (88/100) — 15ビート中14が適切な位置に存在
    アーク明度: A (92/100) — Cinderella型に明確に一致
    ペーシング: B (74/100) — Catalystがやや遅い（15%、標準は10-12%）
    引き:     B (80/100) — Opening Imageのインパクトが弱い

    【改善提案】
    1. Catalystを10%付近に前倒しすると、第1幕のテンポが改善します
    2. Opening Imageにもう少し強い視覚的フックを...

    [ストーリーマップ画像を表示]
```

#### 開発モード — 新しいストーリーの構造を設計する

```
ユーザー: 復讐劇の映画をCinderella型で構成したい

AI: いくつか確認させてください。
    1. ジャンルは復讐劇ですね。サスペンス寄り？アクション寄り？
    2. ログラインはありますか？
    3. 主要キャラクターは？

ユーザー: サスペンス寄り。ログラインは「夫に裏切られた女が、
         完璧な計画で全てを取り返す」

AI: Cinderella型 × サスペンス復讐劇のビートシートを生成します。

    【Opening Image (0-1%)】感情: -0.3
      → 幸せそうな家庭。しかし微かな違和感。
    【Theme Stated (5%)】感情: -0.2
      → 友人「あなたはもっと自分を大切にすべき」
    【Catalyst (10-12%)】感情: +0.1
      → 夫の裏切りの決定的証拠を発見。
    ...

    [ストーリーマップ画像を表示]
```

## 対応フォーマット

| フォーマット | ビート構成 | 特記事項 |
|---|---|---|
| 映画 | STC 15ビート完全版 | 単一アーク、90-120分 |
| ドラマシリーズ | マクロアーク + エピソード8ビート | クリフハンガー設計、A/B/Cストーリー |
| ショートドラマ | マクロアーク + 超圧縮4ビート | 冒頭3秒フック、1-3分/話 |
| マンガ | 巻単位STC + 話単位起承転結 | ページめくり演出、見開き設計 |

## 6つの感情アーク

| 型 | 英語名 | パターン | 代表作 |
|---|---|---|---|
| 成功 | Rags to Riches | ↗ 上昇 | フォレスト・ガンプ |
| 悲劇 | Tragedy | ↘ 下降 | ロミオとジュリエット |
| 逆転 | Man in a Hole | ↘↗ 下降→上昇 | ダイ・ハード |
| 失落 | Icarus | ↗↘ 上昇→下降 | ソーシャル・ネットワーク |
| 感動 | Cinderella | ↗↘↗ 上昇→下降→上昇 | ロッキー |
| 皮肉 | Oedipus | ↘↗↘ 下降→上昇→下降 | ファイト・クラブ |

## 評価カテゴリ

| カテゴリ | 評価内容 |
|---|---|
| **構造** | Save the Catビートの充足率と配置 |
| **アーク明度** | 6型への一致度、感情曲線の明確さ |
| **ペーシング** | ビートのタイミング、フォーマット基準との照合 |
| **引き** | フォーマット固有のフック（冒頭、クリフハンガー等） |

## 手動で使う（Claude Code不要）

可視化スクリプトは単体でも使えます。

```bash
# JSONを作成して可視化
python scripts/visualize_story.py --config my_story.json --output story_map.png
```

JSONの構造は `sample_analysis.json` を参照してください。
